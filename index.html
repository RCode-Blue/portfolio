Resource Scheduler

React | Django | AntD

Resource Scheduler is an app for resource booking. It designed to cater for more than one organisation.

I came up with the idea for this project following a comment by a friend who whished that there was an app available for reserving meeting rooms online. I decided to learn Django and try to create a simple web application using what I learned.

...

Features:

  • Users can create / edit organisations and resources
  • Users can be assigned to organisations
  • User views are filtered according to organisations they are linked to, and their resources
  • When booking resources, reserved days and times are filtered out to avoid clashes

Notes on demo site:

  • No permission filtering - All functionality is available
  • The top menu are links to "admin" level pages - All bookings, users, resources and organisations are available here
  • The "breadcrumb" menu are links to "user-level" pages - Bookings made by the user and resources available to them are available here

Onboarding

React | Flask | Materialize

Borne out of a need to manage a growing business, Onboarding is an app created to manage onboarding tasks for new employees. The application provides for the creation of individualised task templates to cater for different sites and job titles.

Tasks can be created by managers, then assigned to templates and organised in a sequence. New employees are assigned a template and an onboarding buddy. The new employee logs in and based on their account, is able to view task templates assigned to them. When done, tasks are ticked off and the completion date is then marked against it.

...

Features:

  • Individual task creation with instructor name
  • Tasks can be edited after creation
  • Template creation with custom task sequencing
  • Users are assigned task templates
  • Once assigned a template, user tasks do not change even if the tasks definitions are edited

Demo site notes:

  • Webpage has been configured to automatically load a demo user with an existing profile
  • It runs on a free Heroku dyno which goes to sleep when not used. You may see an error message when first loading the webpage. Please wait a few minutes before loading again.
  • No permission filtering. All pages are viewable with edit permissions

bShare

A bike sharing application.

Want to explore Sydney? Don't want to walk? With bShare, you don't have to. For only $2 per hire, you can pick up a bike at any station, check out the harbour city, then return it at the nearest station.

Powered by Rails, this application keeps track of where each bike is located. Users check out a bike from any station, then pays by credit card upon return. Credit card payment is done using Stripe.

Q-Mailer

A survey web application.

Users are able to create custom surveys and send an email to a list of recipients. Users login using oAuth to Google+. Credit card payment is via Stripe API, data is stored in MongoDB and SendGrid is used for sending emails.

My motivation for this project is the opportunity to work with the MERN stack as well as the required APIs. It was a steep learning experience as most of the technologies involved was new to me. However I found the learning process to be highly rewarding.

Node | React | Redux
Stripe | SendGrid | oAuth | MongoDB
